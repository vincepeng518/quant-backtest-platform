## 1. 快速開始

### 1.1 啟動網頁

```bash
cd /home/user/backtest_web
python3 -m streamlit run app.py --server.port=8765 --server.headless=true
```

打開瀏覽器訪問 `http://localhost:8765`

### 1.2 第一次跑回測（3 步驟）

| 步驟 | 操作 | 說明 |
|------|------|------|
| 1 | 點左側 **「一鍵測試資料」** | 不用網路即可生成 500 根 BTC 模擬 K 線 |
| 2 | 點 **「▶️ 執行回測」** | 自動用預設的 SMA 交叉策略回測 |
| 3 | 切到 **Overview / Performance Summary / Charts / 蒙地卡羅** | 查看不同維度的結果 |

整個流程不到 1 分鐘就能看到完整的回測報告。

## 3. 資料來源

在 Sidebar **「資料來源」** 選 3 選 1：

### 3.1 加密貨幣 (CCXT) — 預設

從交易所抓取真實 K 線資料。

| 欄位 | 說明 | 預設值 |
|------|------|--------|
| 選擇資料來源 | 加密貨幣 (CCXT) | ✓ |
| 交易所 | BingX / Binance / OKX / Bybit / Gate.io / Kraken | BingX |
| 交易對 | 格式：BTC/USDT、ETH/USDT 等 | BTC/USDT |
| BingX 熱門交易對 | 點 expander 可快速填入熱門幣種 | — |
| 時間框架 | 1m / 5m / 15m / 30m / 1h / 2h / 4h / 6h / 12h / 1d / 1w | 1h |
| 回看天數 | 7 ~ 1825 天 | 180 |

**操作步驟：**
1. 選擇交易所（推薦 BingX）
2. 輸入或從「熱門交易對」選幣種
3. 選時間框架
4. 設定回看天數
5. 點 **「🔄 抓取資料」**

**注意事項：**
- 需網路連線
- 部分交易所的 API 可能有 rate limit

### 3.2 上傳 CSV

上傳自己的 K 線資料，CSV 需包含以下欄位：

```
timestamp,open,high,low,close,volume
2024-01-01 00:00:00,42000,42500,41800,42300,1234.5
2024-01-01 01:00:00,42300,42600,42200,42500,1100.2
...
```

支援時間戳記或 datetime index 格式。

### 3.3 配對交易 (Pair)

兩種標的同時下反向部位的統計套利策略。

| 預設組合 | 標的 1 | 標的 2 |
|----------|--------|--------|
| BTC/ETH 比率配對 | BTC/USDT | ETH/USDT |

**操作步驟：**
1. 選「配對交易 (Pair)」
2. 從下拉選單選配對組合
3. 設定交易所、時間框架、回看天數
4. 點 **「🔄 抓取配對資料」**

策略代碼必須回傳 **4 個 series** (long_entries, long_exits, short_entries, short_exits)，詳見 13. 撰寫自訂策略。

### 3.4 一鍵測試資料（推薦新手）

不需網路，**生成 500 根固定 seed 的模擬 K 線**（基於幾何布朗運動模擬 BTC 價格，固定 seed=42，每次結果一致）。

**用於：**
- 快速體驗系統功能
- 比較不同策略的相對表現
- 開發和測試

## 4. 策略管理

在 Sidebar **「策略管理」** 點 expander 展開，3 種方式新增策略：

### 4.1 上傳 .py 檔案

1. 展開「上傳 / 貼上策略」

- 等內建策略範本自動載入到「我的策略庫」

### 4.5 策略庫使用

主內容區「選擇策略來源」下拉選單包含：
- 「（自訂）」

- 我的策略庫（分組「── 我的策略庫 ──」）

選擇後自動載入策略代碼。

## 7. 結果頁面（5 大分頁）

回測完成後，切換下方 5 個子分頁查看結果。

### 7.1 Overview

**TradingView Strategy Tester 風格** 的總覽頁面。

#### 7.1.1 頁面標題列

- 標題：**Strategy Backtest Report**
- 副標題：日期範圍、交易次數、初始資金
- 右側控制：
  - **Buy & hold** checkbox：是否顯示 Buy & Hold 基準線
  - **Absolute / Percentage** toggle：金額 vs 百分比

#### 7.1.2 5 大核心 KPI

每個 KPI 卡片有 ⓘ icon（hover 看說明）：

| KPI | 說明 | 解讀 |
|-----|------|------|
| **Total P&L** | 總損益（金額 + 百分比）| 正數=獲利，負數=虧損 |
| **Max equity drawdown** | 最大資金回撤（金額 + 百分比）| 越小越好，< 20% 為良好 |
| **Total trades** | 總交易次數 | — |
| **Profitable trades** | 獲利交易數 / 總數 + 勝率 | > 50% 為良好 |
| **Profit factor** | 毛利 / 毛損 | > 1.5 為良好，> 2 為優秀 |

#### 7.1.3 6 個次要 KPI

| KPI | 說明 | 公式 |
|-----|------|------|
| **Buy & Hold** | 基準報酬率 | 不交易單持有的報酬 |
| **α 超額報酬** | 策略 - B&H | 正數代表跑贏持有 |
| **Sharpe** | 風險調整報酬 | (平均報酬 - 無風險利率) / 標準差，> 1 為良好 |
| **Sortino** | 下行風險調整 | 只看虧損波動，> 1 為良好 |
| **CAGR** | 年化報酬率 | 年複合成長率 |
| **Calmar** | CAGR / MaxDD | > 1 為良好 |

**主圖（72% 高度）**：

- **紅/綠 equity 曲線**：spline 抗鋸齒
- **橘色 Buy & Hold 虛線**：對照基準

- **橘色叉**：出場點

#### 7.1.5 Trade Statistics（4×4 網格）

## 8. 自動參數優化

### 8.1 流程

1. 選擇策略
2. 設定優化模式（Bayesian / Grid / Random）
3. 設定優化目標
4. 設定參數空間（哪些參數要優化）

### 8.2 優化模式

| 模式 | 適用 | 預設試驗數 |
|------|------|------------|
| **Bayesian (Optuna)** | 參數空間大、需要高效搜尋 | 50 |
| **Grid Search** | 參數少、候選值明確 | 全部組合 |
| **Random Search** | 基準線 | 50 |

### 8.3 優化目標（Objective）

從下拉選單選擇（list_objectives）：
- `sharpe_ratio`（預設）
- `total_return_pct`
- `calmar_ratio`
- `sortino_ratio`
- `profit_factor`
- `max_drawdown_pct`
- `win_rate`
- `avg_pnl_pct`
- `cagr`

### 8.4 參數空間（兩種編輯器）

切換 `輸入` / `模式` tab：

**輸入 tab**：固定參數（每個策略都要用的常數）

**模式 tab**：
- **Bayesian 模式**：範圍型（每個參數 = 名稱 + 型態 + 範圍）
  - 支援 int / float / float_log / categorical
  - 顯示「搜尋空間（估）」+「要測試」+「搜尋效率」
- **Grid / Random 模式**：候選值清單型（每個 key 是參數名，value 是清單）

## 9. Walk-Forward 驗證

切到 **「Walk-Forward 驗證」** 分頁（頂部第三個 tab）。

### 9.1 概念

將資料切成多個 in-sample（訓練）和 out-of-sample（測試）區段：
```
|---Train---|--Test--|---Train---|--Test--|---Train---|--Test--|
```

驗證策略在「未見過的資料」上也能獲利，避免過擬合。

### 9.2 設定

| 參數 | 預設值 | 說明 |
|------|--------|------|
| 切分數量 | 5 | 切幾段 |
| 訓練集佔比 | 0.7 | 訓練佔 70% |
| 錨定窗口 | ✗ | 勾選則從頭開始（否則滑動窗口） |
| 優化目標 | sharpe_ratio | 內部優化的目標 |
| 內部優化器 | Optuna (Bayesian) | 每次 fold 內部優化用 |
| 每次 trials | 20 | Optuna 跑幾次 |

### 9.3 結果

#### 9.3.1 綜合 OOS 表現（5~6 個 KPI）

OOS = Out-of-Sample（測試集）。包括：OOS 交易數、總報酬、勝率、平均損益、最大單筆虧損。

#### 9.3.2 過擬合風險評估

- 平均訓練指標 vs 平均測試指標
- **指標衰退率**：< 30% 為低風險，30~60% 中風險，> 60% 高風險

#### 9.3.3 參數穩定度分析

- 穩定度評分（0~100）
- 每個參數的「平均最佳值」、「標準差」、「變異係數 CV」、「所有最佳值」

#### 9.3.4 各區段詳細結果

表格顯示每個切分區段的：訓練範圍、測試範圍、訓練指標、測試指標、測試報酬/回撤/勝率/交易數。

## 10. 蒙地卡羅模擬

在 **Overview / Performance Summary / List of Trades / Charts / 蒙地卡羅** 中的「蒙地卡羅」分頁。

### 10.1 概念

透過**隨機重排交易順序** N 次，模擬「如果未來交易以不同順序發生」的各種可能結果，評估策略的**穩健性**。

### 10.2 設定

| 參數 | 預設 | 說明 |
|------|------|------|
| 模擬次數 | 1000 | 100~10000 |
| 抽樣方法 | shuffle (重排) | shuffle = 重排 / bootstrap = 有放回 |
| 破產門檻 (%) | 50 | 虧損超過此 % 視為破產 |

### 10.3 結果

#### 10.3.1 5 個 KPI

- **中位數報酬** + 平均
- **最壞 5%** + 最好 5%
- **中位數回撤** + 最壞 5% 回撤
- **破產機率**（> 10% 高風險，5~10% 中風險，< 5% 低風險）

## 13. 撰寫自訂策略

### 13.1 基礎格式

每個策略必須定義 `generate_signals(df, params)` 函數：

```python
def generate_signals(df, params):
    """
    參數:
        df: K 線 DataFrame（含 open, high, low, close, volume）
        params: dict 策略參數
    回傳:
        (entries, exits) 或 (entries, exits, None, long_entries, long_exits, short_entries, short_exits)
    """
```

### 13.2 簡單範例：均線交叉

```python
def generate_signals(df, params):
    fast = params.get('fast_period', 20)
    slow = params.get('slow_period', 50)
    
    df['sma_fast'] = df['close'].rolling(fast).mean()
    df['sma_slow'] = df['close'].rolling(slow).mean()
    
    # 黃金交叉：快線上穿慢線 → 進場
    entries = (df['sma_fast'] > df['sma_slow']) & (df['sma_fast'].shift(1) <= df['sma_slow'].shift(1))
    # 死亡交叉：快線下穿慢線 → 出場
    exits = (df['sma_fast'] < df['sma_slow']) & (df['sma_fast'].shift(1) >= df['sma_slow'].shift(1))
    
    return entries, exits
```

### 13.3 進階：支援多空（long_short）

```python
def generate_signals(df, params):
    # ... 計算指標 ...
    
    long_entries = (df['sma_fast'] > df['sma_slow']) & (df['sma_fast'].shift(1) <= df['sma_slow'].shift(1))
    long_exits = (df['sma_fast'] < df['sma_slow']) & (df['sma_fast'].shift(1) >= df['sma_slow'].shift(1))
    
    short_entries = (df['rsi'] > 70) & (df['rsi'].shift(1) <= 70)  # RSI 超買 → 做空
    short_exits = (df['rsi'] < 50) & (df['rsi'].shift(1) >= 50)  # RSI 中線 → 平倉
    
    # 簡單回傳也支援（會自動轉 long-only）
    entries = long_entries
    exits = long_exits
    
    return entries, exits, None, long_entries, long_exits, short_entries, short_exits
```

## 14. 常見問題 FAQ

### Q1: 為什麼「執行回測」後沒有結果？

可能原因：
- 策略代碼錯誤（看警告訊息）
- 策略沒有產生任何進場訊號（看「⚠️ 策略沒有產生任何進場訊號」警告）
- 資料筆數太少（< 50 根通常無法產生有效訊號）

### Q2: 蒙地卡羅的「破產機率」怎麼解讀？

破產機率 = 模擬中權益跌破破產門檻（預設 50%）的比例。
- < 5%：低風險
- 5-10%：中風險
- > 10%：高風險

但要注意：破產機率只反映「**單次回測**」的隨機性，不等於實際交易的破產率。

### Q3: Walk-Forward 與自動參數優化有什麼差別？

- **自動參數優化**：在**全部資料**上找最佳參數
- **Walk-Forward 驗證**：模擬**真實交易**（在未見過的資料上測試）

Walk-Forward 更接近「實際上線後的表現」，是評估策略**泛化能力**的標準方法。

### Q4: 為什麼優化時顯示「沒有找到任何有效組合」？

可能原因：
- 參數空間太大（組合爆炸）
- 策略代碼在某個參數組合下出錯
- 數據不足以產生有效訊號

建議：縮小參數範圍，或先用 Grid Search 跑小規模測試。

### Q5: 如何匯入外部資料？

兩種方式：
- **CSV 上傳**：在 Sidebar 選「上傳 CSV」→ 上傳檔案
- **配對交易**：從交易所抓取（需網路）

### Q6: 切換主題後圖表顏色沒變？

- 點 FAB 按鈕右側的 `<html data-theme>` 已切換
- 圖表使用 `var(--plotly-paper)` 等 CSS 變數
- **如果圖表顏色沒更新**：可能是 streamlit 渲染延遲，重新整理頁面（Ctrl+R）即可

### Q7: 怎麼清空所有資料重新開始？

1. Sidebar → 點「清除資料」
2. 或重新整理頁面

### Q8: 為什麼優化跑很久沒結果？

- **Optuna 預設 50 trials** — 通常幾秒到幾分鐘
- **Grid Search 組合爆炸** — 例如 5 個參數各 10 個值 = 10^5 = 100,000 次回測
- 建議先用小範圍測試

### Q9: Buy & Hold 線是什麼？

Buy & Hold（買入持有）= 在回測期間一開始就買入標的，期間完全不交易。
- 用來對比「策略 vs 什麼都不做」
- 比較 alpha（超額報酬）即可知道策略的價值

### Q10: 免責聲明

> ⚠️ 免責聲明：本工具僅供研究與教育用途。回測結果不代表未來表現。

加密貨幣交易有高風險，過去的回測表現**不能保證**未來收益。請謹慎投資，勿投入無法承受虧損的資金。
